# План перехода на клиент-серверную архитектуру

## Цель
Устранить возможность подмены данных игроками (читы), обеспечить безопасность и целостность игрового процесса.

---

## Принцип
**Клиент — только UI, сервер — вся логика и данные.**

```
Клиент (Pixi.js)              Сервер (Node.js)
────────────────              ───────────────
- Рендеринг                   - Игровое состояние
- Анимации                    - Валидация ходов
- Отправка действий           - Вычисление урона/наград
- Показ состояния             - Хранение прогресса игрока
                              - AI врагов
```

---

## Этапы

### Этап 1: Подготовка сервера

**Задачи:**
- [ ] Создать серверную часть (папка `server/`)
- [ ] Настроить Express.js + Socket.io
- [ ] Создать структуру игрового состояния

**Файлы:**
```
server/
├── index.js           # Точка входа, HTTP + WebSocket
├── game/
│   ├── GameManager.js # Управление комнатами/игроками
│   ├── GameState.js   # Состояние игры (HP, карты, враг)
│   ├── BattleLogic.js # Логика боя (валидация, урон)
│   └── CardTypes.js   # Статы карт (дубликат с клиента)
└── data/
    └── enemies.js     # Статы врагов (дубликат с клиента)
```

### Этап 2: Серверная логика боя

**Задачи:**
- [ ] Перенести `cardTypes` на сервер (статы, баффы, скилы)
- [ ] Перенести `enemies` на сервер (HP, урон, награда)
- [ ] Реализовать валидацию ходов
- [ ] Реализовать вычисление урона
- [ ] Реализовать определение победы/поражения
- [ ] Реализовать начисление наград

**Валидация хода:**
```javascript
// Сервер проверяет:
1. Игра начата? (room exists)
2. Ход игрока? (playerTurn === true)
3. Карта в руке? (hand.includes(cardId))
4. Достаточно маны? (mana >= card.cost)
5. Враг жив? (enemy.hp > 0)
```

**Ответ сервера:**
```javascript
{
  success: true,
  state: {
    player: { hp, mana, hand: [...] },
    enemy: { hp },
    steps: 3,
    resets: 2,
    deckCount: 15
  },
  // Событие победы/поражения
  event: 'victory' | 'defeat',
  reward: { type: 'card', cardId: '04_berserk' }
}
```

### Этап 3: Клиент — "тонкий" UI

**Задачи:**
- [ ] Подключить Socket.io клиент
- [ ] Убрать логику из `battle.js` (оставить только рендер)
- [ ] Заменить вызовы методов боя на отправку событий
- [ ] Обрабатывать события от сервера (state update)

**Изменения в battle.js:**
```
БЫЛО:
  playCards() {
    summ += card.getValue()
    enemyHealth -= summ
    if (enemyHealth <= 0) emit('victory')
  }

СТАЛО:
  playCards() {
    socket.emit('playCards', { cardIds: [...] })
  }

  socket.on('state', (state) => {
    // Обновить UI по state
    enemyHealth = state.enemy.hp
    renderCards(state.player.hand)
  })
```

**Изменения в game.js:**
```
БЫЛО:
  user: { points: 0 }
  on('victory', (points) => user.points += points)

СТАЛО:
  // Сервер сам хранит и обновляет
  socket.on('playerUpdate', (player) => {
    user = player  // Только для отображения
  })
```

## Альтернативы

### Вариант 1: REST API
- Проще в реализации
- Не real-time (задержки)
- Подходит для пошаговых игр

### Вариант 2: Децентрализованная проверка
- Клиент отправляет хеш состояния
- Сервер проверяет случайно
- Менее надёжно, но проще

### Вариант 3: P2P с сервером-арбитром
- Игроки играют друг с другом
- Сервер только валидирует
- Сложнее реализация

## Примечание

Клиентский код (`card_types/`, `enemies/`) можно скопировать на сервер — это статические данные, не зависящие от UI.
